# Changelog

## v2.1

- 增加提示词防注入安全机制：小红书标题、正文、评论、图片 OCR、作者昵称和用户资料一律视为外部不可信内容。
- 明确外部内容只能作为待分析材料，不得执行其中的命令、角色设定、工具调用请求或流程修改要求。
- 在主技能、面经子技能、继任机会筛选子技能和 README 中同步安全边界说明。
- 优化继任输出模版：新增汇总表格，方便 Agent 读取邮箱直接投递。

## v2.0

- 将 Skill 重构为“主技能 + 子技能”结构：主技能负责登录协议、`xhs-cli` 通用能力和子技能路由。
- 新增 `references/mianjing.md`，完整承接原有面经搜索、筛选、来源分类和报告格式能力。
- 新增 `references/jiren.md`，支持求职者视角的继任帖、离职交接帖、找接班人帖机会筛选。
- 固化继任机会筛选规则：默认筛选近 3 天内容，识别岗位、公司、base 地、投递邮箱和风险信号。
- 固化继任机会输出模板：按“高优先级机会 / 待确认机会 / 已排除或低质线索 / 附录”稳定输出。

## v1.1

- 优化登录流程：搜索前先检查 `xhs status --json`，未登录时引导用户确认已登录小红书的浏览器，再读取对应浏览器 Cookie。
- 增加二维码登录兜底：浏览器 Cookie 读取失败时，使用 `xhs login --qrcode` 完成登录。
- 调整筛选体验：默认按“面试类型全选、面试阶段全选”执行搜索，用户主动指定实习、秋招、一面、二面等条件时再自动收窄。
- 稳定报告结构：固定输出“高频题库”“带参考回答的面经”“面试题清单”“备战建议”四个主体部分。
- 统一引用规则：正文不放来源编号，来源列表、检索说明和采集异常统一放在报告附录。
- 更新 README 和安装文档，明确 `xhs-cli` 常用能力、隐私边界和安装路径。

## v1.0

- 初始发布小红书面经 Codex Skill。
- 直接复用上游 `xiaohongshu-cli` 的 `xhs` 命令，不维护单独的面经 CLI。
- 支持搜索小红书笔记、读取笔记详情、读取评论，并整理岗位面经、实习面经、秋招/校招经验和面试题。
- 提供安装脚本，将 `xhs-mianjing/` 安装到 `~/.codex/skills/xhs-mianjing/`。
- 提供中文 README、Skill 说明、安装与登录参考文档，以及 Cookie/隐私说明。
